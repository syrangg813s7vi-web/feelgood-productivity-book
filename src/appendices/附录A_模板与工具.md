# 附录A：模板与工具

> *"工具本身不会让你变得高效，但正确的工具可以让你更容易保持高效。"*
> ——基于《Feel-Good Productivity》核心理念

---

## A.1 GPS目标设定工作表

### 用途
将模糊的愿望转化为可执行的系统，避免"目标设定陷阱"。

### 模板

```
=== GPS目标设定工作表 ===
日期：______________

【Goal - 目标】
我想达成：_______________________________________
（具体、可衡量的结果）

为什么这个目标对我重要？
1. _______________________________________
2. _______________________________________
3. _______________________________________

成功标准是什么？（如何知道我达成了？）
_______________________________________

---

【Plan - 计划】
反向工程：从目标倒推里程碑

达成目标的日期：______________

最后30天需要完成：_______________________________
最后60天需要完成：_______________________________
最后90天需要完成：_______________________________

【System - 系统】
每周/每天我需要做什么？

每周固定行动：
1. _______________________________________
2. _______________________________________
3. _______________________________________

每天固定行动：
1. _______________________________________
2. _______________________________________

环境设计：
- 我需要移除的障碍：_______________________________
- 我需要添加的触发器：_____________________________
- 我需要的工具/资源：_____________________________

潜在失败点预判：
如果出现________________情况，我会__________________对策
如果出现________________情况，我会__________________对策

每周审查时间：每周_____  _____:_____
```

### 使用指南
1. **填写Goal**: 用SMART原则（具体、可衡量、可达成、相关、有时限）
2. **完成Plan**: 从终点倒推，分解为90天/60天/30天里程碑
3. **设计System**: 关注环境设计而非意志力
4. **每周审查**: 在固定时间回顾进展，调整系统

1. **每12周为一个周期**：一年可以完成4个周期
2. **最多3个目标**：避免分散注意力
3. **每周评分**：在固定时间（如周日晚上）完成
4. **85%原则**：目标不是100%完美，而是85%以上的持续执行

## A.4 周问责会议（WAM）议程

### 用途
每周固定的自我问责会议，确保12周目标持续推进。

### 议程模板

```
=== 每周问责会议（WAM） ===
时间：每周_____  _____:_____
时长：20-30分钟

【第一步：回顾上周承诺】（5分钟）
上周我的承诺是：
1. _____ 是否完成？□是 □否
2. _____ 是否完成？□是 □否
3. _____ 是否完成？□是 □否
4. _____ 是否完成？□是 □否
5. _____ 是否完成？□是 □否

上周执行率：_____%

【第二步：庆祝胜利】（3分钟）
本周做得好的3件事（无论多小）：
1. _______________________________________
2. _______________________________________
3. _______________________________________

我要为这3件事感到自豪！

【第三步：分析差距】（5分钟）
如果执行率<85%：
- 哪些承诺没有完成？_______________________
- 为什么没完成？（诚实分析）
  □ 目标太庞大 → 拆分得更小
  □ 时间不够 → 重新评估优先级
  □ 系统失效 → 设计新系统
  □ 意志力不足 → 环境设计不足
  □ 其他：_________________________________

从这次失败中，我学到什么？
_______________________________________

【第四步：设定本周承诺】（5分钟）
本周我承诺完成（3-5个具体行动）：
1. _______________________________________
2. _______________________________________
3. _______________________________________
4. _______________________________________
5. _______________________________________

本周我的焦点是：___________________________

【第五步：系统检查】（5分钟）
我的系统是否需要调整？
- 时间管理：□保持 □调整
- 环境设计：□保持 □调整
- 工具使用：□保持 □调整
- 能量管理：□保持 □调整

具体调整计划：
_______________________________________

【第六步：下确认】（2分钟）
我承诺本周完成上述行动。
我会记录每天的进展。
下周五同样时间，我会向自己汇报。

签字（心理）：_________________
```

### 使用指南
1. **固定时间**：每周同一时间，建立仪式感
2. **诚实评分**：不要给自己找借口，85%就是及格线
3. **庆祝胜利**：即使是小进步也值得认可
4. **系统思维**：失败时问"系统哪里出了问题"，而不是"我哪里错了"

## A.7 PARA分类法快速参考卡

### 用途
快速参考PARA方法，组织你的数字信息。

### PARA四类

**Projects（项目）**
- 定义：你正在主动从事的短期努力（可在几天到几个月内完成）
- 判断标准：有明确的完成标准和截止日期
- 示例：
  - "完成2024年度报告" ✓
  - "学习Python" ✗ （这是一个Area）
- 项目数量建议：同时不超过5-8个

**Areas（领域）**
- 定义：你有持续责任的长期领域（没有完成日期）
- 判断标准：如果你停止关注，会出问题
- 示例：
  - 健康、财务、职业发展、人际关系
  - "学习"是一个Area，"完成Python课程"是一个Project

**Resources（资源）**
- 定义：你可能在未来用到的主题或兴趣
- 判断标准：你暂时在做，但不想丢失
- 示例：
  - 有趣的文章、影像资料、书籍笔记
  - 可能未来的项目灵感
  - 研究材料

**Archives（归档）**
- 定义：已完成或不再活跃的项目和领域
- 判断标准：已经完成或暂停
- 示例：
  - 2023年的已完成项目
  - 之前的工作职位相关材料

### 快速决策树

```
新信息来了，问自己三个问题：

1. 这个信息是否需要我采取行动？
   是 → 它可能是一个Project
   否 → 继续

2. 这是否是我有持续责任的领域？
   是 → 它可能是一个Area
   否 → 继续

3. 我是否会在未来某个时候用到它？
   是 → 放入Resources
   否 → 如果有价值，归档到Archives
       否，删除
```

### 实施检查清单

- [ ] 列出你当前的所有Projects（检查每个是否有截止日期）
- [ ] 识别你的Areas（5-10个）
- [ ] 创建Resources文件夹，按兴趣分类
- [ ] 将旧项目移到Archives
- [ ] 设置每周审查：检查Projects进度和Areas健康度

### 常见错误

❌ 过度分类：信息太多类别，导致混乱
✅ 只有4类：Projects, Areas, Resources, Archives

❌ Project和Area不分：把"健康"当作Project
✅ "健康"是Area，"完成30天运动挑战"是Project

❌ 从不归档：Projects文件夹越来越大
✅ 每月审查，将完成的项目移到Archives

## A.8 生命之轮评估工具

### 用途
全面评估你的人生各个领域，找到需要平衡的地方。

### 生命之轮（9+1个领域）

```
         职业(1)
           ^
           |
    人际(7) |  财务(2)
      \    |    /
       \   |   /
    家庭(8) | 个人成长(3)
         \ | /
    环境(9)● 健康(4)
         /| \
    /   |   \
   /    |    \
娱乐(10) |   贡献(5)

        |

   分数：_____

   说明：_____________________________________

   分数：_____

   分数：_____

   分数：_____
   说明：_____________________________________

   分数：_____
   说明：_____________________________________

   分数：_____
   说明：_____________________________________

   分数：_____
   说明：_____________________________________

    分数：_____
    说明：_____________________________________

【分析】

1. _____ (____分)
2. _____ (____分)
3. _____ (____分)

我的最低3个领域：
1. _____ (____分)
2. _____ (____分)
3. _____ (____分)

【可视化】
（将分数画在生命之轮上，看看你的轮子是圆的还是有凹凸？）

【30天行动计划】
选择1个最低领域，写下1个具体的改进行动：
领域：_______________
行动：_____________________________________
开始时间：_______  预期结果：________________
```

### 使用指南
1. **诚实评分**：这是给你自己看的，不需要伪装
2. **画出来**：视觉化更能看到不平衡
3. **关注最低领域**：提升最低领域比提升最高领域更有幸福感收益
4. **90天评估一次**：不是天天看，季度回顾足够

### 生命之轮警告信号

- ⚠️ 某个领域<4分：需要立即关注
- ⚠️ 最高分和最低分差距>6分：严重失衡
- ⚠️ 连续3次评估某个领域持续下降：需要深度反思

---

**📌 使用这些模板的原则**

1. **从简单开始**：不要一次用所有模板，选一个先开始
2. **保持一致性**：每周固定时间使用，形成习惯
3. **诚实记录**：数据的价值在于真实，不在于好看
4. **定期审查**：每月问"这些模板是否还帮我？"
5. **个性化调整**：这些是起点，不是终点，根据需要修改

---

**如果你不知道从哪个模板开始，建议：**
