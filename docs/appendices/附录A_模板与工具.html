A.7 PARA分类法快速参考卡

用途

快速参考PARA方法，组织你的数字信息。

PARA四类

Projects（项目）

  • 定义：你正在主动从事的短期努力（可在几天到几个月内完成）
  • 判断标准：有明确的完成标准和截止日期
  • 示例：
    • "完成2024年度报告" ✓
    • "学习Python" ✗ （这是一个Area）
  • 项目数量建议：同时不超过5-8个

Areas（领域）

  • 定义：你有持续责任的长期领域（没有完成日期）
  • 判断标准：如果你停止关注，会出问题
  • 示例：
    • 健康、财务、职业发展、人际关系
    • "学习"是一个Area，"完成Python课程"是一个Project

Resources（资源）

  • 定义：你可能在未来用到的主题或兴趣
  • 判断标准：你暂时在做，但不想丢失
  • 示例：
    • 有趣的文章、影像资料、书籍笔记
    • 可能未来的项目灵感
    • 研究材料

Archives（归档）

  • 定义：已完成或不再活跃的项目和领域
  • 判断标准：已经完成或暂停
  • 示例：
    • 2023年的已完成项目
    • 之前的工作职位相关材料

快速决策树

新信息来了，问自己三个问题：

1. 这个信息是否需要我采取行动？
   是 → 它可能是一个Project
   否 → 继续

2. 这是否是我有持续责任的领域？
   是 → 它可能是一个Area
   否 → 继续

3. 我是否会在未来某个时候用到它？
   是 → 放入Resources
   否 → 如果有价值，归档到Archives
       否，删除

实施检查清单

  • 列出你当前的所有Projects（检查每个是否有截止日期）
  • 识别你的Areas（5-10个）
  • 创建Resources文件夹，按兴趣分类
  • 将旧项目移到Archives
  • 设置每周审查：检查Projects进度和Areas健康度

常见错误

❌ 过度分类：信息太多类别，导致混乱 ✅ 只有4类：Projects, Areas, Resources, Archives

❌ Project和Area不分：把"健康"当作Project ✅ "健康"是Area，"完成30天运动挑战"是Project

❌ 从不归档：Projects文件夹越来越大 ✅ 每月审查，将完成的项目移到Archives

使用指南

  1. 诚实评分：这是给你自己看的，不需要伪装
  2. 画出来：视觉化更能看到不平衡
  3. 关注最低领域：提升最低领域比提升最高领域更有幸福感收益
  4. 90天评估一次：不是天天看，季度回顾足够

生命之轮警告信号

  • ⚠️ 某个领域<4分：需要立即关注
  • ⚠️ 最高分和最低分差距>6分：严重失衡
  • ⚠️ 连续3次评估某个领域持续下降：需要深度反思

📌 使用这些模板的原则

  1. 从简单开始：不要一次用所有模板，选一个先开始
  2. 保持一致性：每周固定时间使用，形成习惯
  3. 诚实记录：数据的价值在于真实，不在于好看
  4. 定期审查：每月问"这些模板是否还帮我？"
  5. 个性化调整：这些是起点，不是终点，根据需要修改

如果你不知道从哪个模板开始，建议：

  • 第1周：GPS目标设定工作表（明确方向）
  • 第2-4周：12周年度规划（建立节奏）
  • 第2周起：每日日记（建立自我认知）
  • 第4周起：每周问责会议（确保执行）
