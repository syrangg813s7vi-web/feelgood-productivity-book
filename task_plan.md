# Task Plan — 全书出版级审核 & 修改（BookReviewer 规范）

## Goal
对全书（`book/BOOK_FULL_Chapter*.md`）按“专业出版编辑”规范进行系统审核，并直接在仓库中完成第一轮可自动化落地的修改；输出可追踪的变更（PR）与复核报告。

## Scope
- 全书章节：`book/BOOK_FULL_Chapter*.md`
- 审核规范：`/root/clawd/agents/BookReviewer/docs/BOOK_REVIEWER_INSTRUCTIONS.md`

## Phases
1. **基线扫描（自动）**（in_progress）
   - 运行 BookReviewer 新规范脚本，生成 reviews_pro 报告
   - 补充：标题层级、跳级、编号、标点/单位、段落过长、列表一致性等统计
2. **自动可修复项批量修改**（todo）
   - 禁词/元叙述残留（若有）：替换/重写句子
   - 标点与符号统一：`10 %`→`10%`，`7 天`→`7天`，中英标点混用等
   - 标题格式统一：清理不规范括号/全角半角/多余副标题（保留信息但统一格式）
   - 列表符号统一（尽量不破坏语义）
3. **结构逻辑/可读性重点修订**（todo）
   - 针对每章：长段落拆分、长句拆句、缺少过渡/总结/FAQ 编号等
   - 需要人工判断部分：以“最小侵入”方式调整
4. **合规性检查**（todo）
   - 裸链、引用来源提示（若出现）
   - 敏感/不当内容扫描
5. **提交与同步**（todo）
   - 建立 `edit/pass-1` 分支提交
   - 创建 PR（包含变更说明 + 复核清单）
   - 同步 Trello 进度（单卡片）
   - 更新 Notion：审核记录/修改摘要

## Decisions
- 修改以“出版级一致性”为准，但不重写章节核心内容；优先修复格式/规范/可读性。
- 一次 PR 做“第一轮编辑通修（pass-1）”；若某章需要大改，再拆分为后续修订 PR。
